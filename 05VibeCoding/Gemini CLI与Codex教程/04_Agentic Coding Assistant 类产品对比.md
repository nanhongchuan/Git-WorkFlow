**问题：** 为什么大模型具体研发的智能开发工具选择推出 CLI 版本，而不是图形化 IDE？


**答：** 这些工具之所以选择以 CLI（命令行界面）形式提供功能。首先是为了适配多操作系统环境（包括无显示器的服务器版 Linux 等场景），它们相比图形化 IDE 壳更轻量，开发与扩展难度较低；同时，由于不依赖特定 UI 框架，功能迭代更快、兼容性更好，这类工具也更易形成活跃的社区生态。
