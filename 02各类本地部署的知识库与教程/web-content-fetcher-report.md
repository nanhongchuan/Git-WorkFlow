# Web-Content-Fetcher Skill 测试报告

## 背景

这篇文章测试了 12 个大模型读链接的能力，结果全部翻车——**没有一个大模型能稳定读懂任意链接里的内容**（幻觉问题严重）。

然后发现了一个 Skill——**web-content-fetcher**，用三种工具的组合解决了这个问题。

---

## 三种工具的分工

| 工具 | 特点 | 限制 |
|------|------|------|
| **Jina Reader** | 过滤广告/导航，只留正文 | 每天 200 次限额（免费） |
| **Scrapling** | 绕反爬，微信公众号都能读 | 无限制但需要配环境 |
| **web_fetch** | Claude 原生工具 | 作为保底方案 |

---

## 测试结果（7 个平台）

| 平台 | 结果 | 备注 |
|------|------|------|
| 微信公众号 | ✅ 满分通过 | - |
| 飞书文档 | ✅ 成功 | 能读懂内容 |
| 小红书 | ✅ 成功 | Jina 被封后自动切换 Scrapling |
| X/Twitter | ✅ 成功 | - |
| CSDN | ✅ 成功 | - |
| 知乎 | ✅ 成功 | - |
| Google Docs | ❌ 失败 | 需登录，无法读取 |

---

## 核心观点

> 这个 Skill 做不到 100% 读取任何网页，但对日常 90% 的网页抓取需求，它给了**稳定、免费、可预期**的解法。

更值得学的是背后的方法论：

> **把现有工具组合起来，设计调度优先级，打包成可复用的 Skill**——遇到同样的问题，你就是那个一个链接搞定所有人的人。
