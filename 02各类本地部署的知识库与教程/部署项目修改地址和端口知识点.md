# 部署项目修改地址和端口的知识点

## 一、配置文件位置

| 文件类型 | 常见字段 |
|----------|----------|
| `config.json` / `config.yaml` | `host`, `port`, `baseUrl` |
| `.env` | `PORT`, `HOST`, `BIND` |
| `package.json` scripts | `--host 0.0.0.0 --port 3000` |
| `vite.config.js` / `webpack.config.js` | `server.port` |
| `docker-compose.yml` | `ports: ["3000:3000"]` |

## 二、host 地址选项及含义

| 配置值 | 访问范围 |
|--------|----------|
| `127.0.0.1` | 仅本机可访问 |
| `localhost` | 仅本机可访问（依赖hosts文件解析，较不稳定） |
| `0.0.0.0` | 本机 + 局域网都可访问 |

## 三、localhost vs 127.0.0.1 的区别

| | 127.0.0.1 | localhost |
|---|---|---|
| 本质 | IP地址（直接可用） | 域名（需DNS解析） |
| 解析 | 不需要查DNS | 查 `/etc/hosts` |
| 稳定性 | 更稳定，不受hosts干扰 | 可能被劫持或IPv6干扰 |
| 常见场景 | 配置文件中使用 | 浏览器地址栏使用 |

## 四、实用建议

- 配置文件里优先写 `127.0.0.1` 而非 `localhost`
- 对外服务用 `0.0.0.0`，本机测试用 `127.0.0.1`
- 修改后记得重启服务才生效
