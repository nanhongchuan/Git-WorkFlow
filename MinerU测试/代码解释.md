# 代码解释

这段Python代码是用于**通过API将本地文件上传到`mineru.net`服务**的一套流程。它采用了常见的“先申请上传链接，再上传文件”的两步走策略。

下面我们来详细解释代码的各个部分：

```python
import requests
```
*   **`import requests`**: 导入Python的`requests`库。这是一个非常流行的库，用于发送HTTP请求（如GET、POST、PUT等），使得与Web服务进行交互变得简单。

```python
token = "官网申请的api token"
```
*   **`token = "官网申请的api token"`**: 这是一个占位符，你需要替换成你在`mineru.net`官网申请到的API访问令牌（Token）。这个Token用于验证你的身份和授权，确保你有权限调用API。

```python
url = "https://mineru.net/api/v4/file-urls/batch"
```
*   **`url = "https://mineru.net/api/v4/file-urls/batch"`**: 这是第一个API请求的目标URL。从路径`file-urls/batch`可以看出，这个接口是用来**批量获取文件上传URL**的。这意味着你先告诉服务器你要上传哪些文件，然后服务器会回复给你一些临时、带签名的上传链接，你再把文件内容传到这些链接上。

```python
header = {
    "Content-Type": "application/json",
    "Authorization": f"Bearer {token}"
}
```
*   **`header`**: 这是一个字典，定义了HTTP请求头信息。
    *   `"Content-Type": "application/json"`: 告诉服务器，请求体（Request Body）的数据格式是JSON。
    *   `"Authorization": f"Bearer {token}"`: 这是认证信息。`Bearer`是一种常见的授权方案，表示后续的`token`是授权凭证。服务器会使用这个`token`来验证你的请求。

```python
data = {
    "enable_formula": True,
    "language": "ch",
    "enable_table": True,
    "files": [
        {"name":"demo.pdf", "is_ocr": True, "data_id": "abcd"}
    ]
}
```
*   **`data`**: 这是一个字典，定义了第一个POST请求的**请求体（Payload）**。它告诉`mineru.net`服务器你打算上传什么文件以及如何处理它们。
    *   `"enable_formula": True`, `"language": "ch"`, `"enable_table": True`: 这些是针对后续文件处理的配置参数，例如是否启用公式识别、文档语言（中文）、是否启用表格识别等。
    *   `"files": [...]`: 这是一个列表，包含你打算上传的文件的信息。
        *   `{"name":"demo.pdf", "is_ocr": True, "data_id": "abcd"}`: 这是关于单个文件`demo.pdf`的描述。
            *   `"name":"demo.pdf"`: 文件的名称。
            *   `"is_ocr": True`: 表示需要对这个文件进行OCR（光学字符识别），通常用于扫描件或图片形式的文档，以便提取文本内容。
            *   `"data_id": "abcd"`: 一个自定义的ID，可能用于在你的系统追踪这个文件，它会被API返回结果关联。

```python
file_path = ["demo.pdf"]
```
*   **`file_path`**: 这是一个列表，存放了本地要上传的文件的路径。这里的`"demo.pdf"`应该与`data["files"][0]["name"]`中的文件名对应。

```python
try:
    response = requests.post(url,headers=header,json=data)
```
*   **`try...except`**: 这是一个错误处理结构，用于捕获可能发生的异常（如网络问题、URL错误等），提高程序的健壮性。
*   **`response = requests.post(url, headers=header, json=data)`**: 这是发送的第一个HTTP请求。
    *   `requests.post()`: 发送一个POST请求。
    *   `url`: 请求的目标是前面定义的获取上传URL的API。
    *   `headers`: 包含认证`token`和`Content-Type`的请求头。
    *   `json=data`: 将`data`字典自动序列化为JSON格式，作为请求体发送。
    *   **目的**: 这个请求的目的是从`mineru.net` API那里获取一个或多个用于实际文件上传的**临时上传链接（Pre-signed URLs）**。

```python
    if response.status_code == 200:
        result = response.json()
        print('response success. result:{}'.format(result))
```
*   **`if response.status_code == 200:`**: 检查HTTP响应状态码。`200`表示请求成功（服务器已接收并处理）。
*   **`result = response.json()`**: 如果请求成功，将服务器返回的JSON响应解析成Python字典。
*   **`print(...)`**: 打印整个响应结果，用于调试。

```python
        if result["code"] == 0:
            batch_id = result["data"]["batch_id"]
            urls = result["data"]["file_urls"]
            print('batch_id:{},urls:{}'.format(batch_id, urls))
```
*   **`if result["code"] == 0:`**: 检查API返回的内部业务状态码。在很多API设计中，`code: 0`通常表示业务逻辑上的成功，即使HTTP状态码是200。
*   **`batch_id = result["data"]["batch_id"]`**: 从API响应中提取批量处理的ID。这有助于你追踪这批文件的上传状态。
*   **`urls = result["data"]["file_urls"]`**: 这一步非常关键！它从API响应中提取出了一批**预授权的上传URL**。每个URL对应一个要上传的文件。

```python
            for i in range(0, len(urls)):
                with open(file_path[i], 'rb') as f:
                    res_upload = requests.put(urls[i], data=f)
                    if res_upload.status_code == 200:
                        print(f"{urls[i]} upload success")
                    else:
                        print(f"{urls[i]} upload failed")
```
*   **`for i in range(0, len(urls)):`**: 遍历之前获取到的所有上传URL。这个循环体是**实际文件上传**的逻辑。
*   **`with open(file_path[i], 'rb') as f:`**: 以二进制读取模式（`'rb'`）打开本地文件。`with`语句确保文件在使用后会被正确关闭。
*   **`res_upload = requests.put(urls[i], data=f)`**: 发送第二个HTTP请求，执行实际的文件上传。
    *   `requests.put()`: 发送一个PUT请求。PUT通常用于向特定URL上传资源或更新资源。
    *   `urls[i]`: 每个文件独有的、临时的上传URL，这些URL通常由云存储服务（如S3、OSS）生成并带有授权信息。
    *   `data=f`: 直接将打开的文件对象作为请求体发送。`requests`库会处理文件的二进制流传输。
    *   **目的**: 将本地文件`file_path[i]`的原始内容上传到`urls[i]`指定的云存储位置。
*   **`if res_upload.status_code == 200:`**: 检查单个文件上传的HTTP状态码。`200`表示上传成功。
*   **`else: print(...)`**: 如果上传失败，打印相应的错误信息。

```python
        else:
            print('apply upload url failed,reason:{}'.format(result.msg))
```
*   **`else: print('apply upload url failed,reason:{}'.format(result.msg))`**: 如果第一个API请求的HTTP状态码是200，但内部业务状态码`result["code"]`不是0，说明申请上传URL的业务逻辑失败了。这里尝试打印`result.msg`作为失败原因。（注意：`result.msg`通常应该写成`result["msg"]`或`result["message"]`，这取决于API实际返回的错误消息字段名）。

```python
    else:
        print('response not success. status:{} ,result:{}'.format(response.status_code, response))
```
*   **`else: print(...)`**: 如果第一个API请求的HTTP状态码不是200（例如400、401、500等），说明请求本身就失败了，打印HTTP状态码和完整的响应对象。

```python
except Exception as err:
    print(err)
```
*   **`except Exception as err:`**: 捕获所有其他未知错误，并打印错误信息。

### 总结流程：

1.  **认证与配置**: 设置API Token、目标API URL、请求头和文件处理配置。
2.  **获取上传链接 (POST请求)**:
    *   向`mineru.net` API发送一个POST请求，告知服务器要上传哪些文件及其处理要求（如OCR、语言）。
    *   这个请求会携带你的API Token进行认证。
    *   服务器验证通过后，会返回一个JSON响应，其中包含每个文件的**临时上传URL**和一个**批量ID**。
3.  **实际文件上传 (PUT请求)**:
    *   遍历服务器返回的每个上传URL。
    *   打开本地对应的文件（二进制模式）。
    *   使用PUT请求将本地文件的二进制内容直接发送到这些临时上传URL。
    *   每个文件上传成功后，API服务会根据之前配置的要求进行后续处理（如OCR、公式识别等）。
4.  **错误处理**: 代码包含多层错误处理，包括HTTP状态码检查和API内部业务状态码检查，以及一个通用的异常捕获，以确保程序能健壮运行。

这是一个典型的文件上传至云服务的API交互模式，通过**分离上传链接的获取和实际文件内容的上传**来增强安全性（临时链接）和灵活性。
