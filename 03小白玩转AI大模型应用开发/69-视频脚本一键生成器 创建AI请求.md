# 69-视频脚本一键生成器 创建AI请求

#### **1. 项目文件与核心函数 (`ai_request.py`)**

*   **文件创建**: 新建一个Python文件，例如命名为 `ai_request.py`，用于集中所有与AI大模型交互的代码。
*   **核心函数**: 定义 `generate_script` 函数，封装整个AI请求逻辑。
    *   **函数目的**: 通过一次调用，即可生成视频的标题和脚本内容。
    *   **接收参数**:
        *   `subject` (视频主题)
        *   `video_len` (视频时长)
        *   `creativity` (视频脚本的创造性，对应模型的 `temperature` 参数)
        *   `api_key` (用户提供的API密钥，不消耗自有Token)
        *   **可选**: `openai_api_base` (如使用课程提供的API密钥，用于指定OpenAI服务地址)
    *   **主要任务**:
        1.  获得视频标题。
        2.  调用维基百科API获取相关信息。
        3.  获得视频脚本内容。

#### **2. 核心库与组件导入**

*   `from langchain.prompts import ChatPromptTemplate`：用于构建与AI交互的结构化提示。
*   `from langchain_openai import ChatOpenAI`：用于连接和使用OpenAI的聊天模型。
*   `from langchain_community.utilities import WikipediaAPIWrapper`：用于调用维基百科API获取信息。

#### **3. 提示模板定义 (`ChatPromptTemplate`)**

*   **创建方式**: 使用 `ChatPromptTemplate.from_messages()` 方法，接收消息列表作为参数。
*   **标题模板 (`title_template`)**:
    *   目的：指导AI根据视频主题生成标题。
    *   结构：消息中包含一个待填入变量 `subject` (视频主题)。
*   **脚本模板 (`script_template`)**:
    *   目的：指导AI综合各项信息生成完整的视频脚本。
    *   结构：消息中包含多个待填入变量：
        *   `title` (AI生成的视频标题)
        *   `duration` (用户提供的视频时长)
        *   `wikipedia_search` (维基百科搜索结果摘要)

#### **4. AI模型定义 (`ChatOpenAI`)**

*   **模型选择**: 使用 `ChatOpenAI` 作为示例模型。
*   **参数配置**:
    *   `openai_api_key`: 赋值为 `api_key` (用户提供的)。
    *   `temperature`: 赋值为 `creativity` (控制生成内容的创造性，0-1之间)。
    *   `openai_api_base`: 可选，用于指定API服务地址。

#### **5. 组装链 (`LCEL`)**

*   **链 (Chain) 概念**: 将提示模板和模型通过 `|` 符号连接起来，形成一个可执行的工作流。
*   **标题链 (`title_chain`)**: `title_template | model` (将标题模板的结果传给模型)。
*   **脚本链 (`script_chain`)**: `script_template | model` (将脚本模板的结果传给模型)。

#### **6. 任务执行流程与数据获取**

*   **A. 获取视频标题**
    *   调用 `title_chain.invoke({"subject": subject})`。
    *   结果为AI消息，通过 `.content` 属性获取具体标题内容，并存入 `title` 变量。

*   **B. 获取维基百科搜索结果**
    *   创建 `WikipediaAPIWrapper` 实例。
    *   设置 `len='zh'` (指定搜索语言为中文，默认英文)。
    *   调用 `instance.run(subject)`，传入视频主题作为搜索词。
    *   获取搜索结果的摘要，存入 `wiki_search_result` 变量。

*   **C. 获取视频脚本**
    *   调用 `script_chain.invoke({"title": title, "duration": video_len, "wikipedia_search": wiki_search_result})`。
    *   结果为AI消息，通过 `.content` 属性获取具体脚本内容，并存入 `script` 变量。

#### **7. 函数返回结果**

*   `generate_script` 函数最终返回：`title` (视频标题), `script` (视频脚本), `wiki_search_result` (维基百科摘要)。

#### **8. 代码测试**

*   在函数外部调用 `generate_script`，传入示例参数（主题、时长、创造性、API Key）。
*   `api_key` 推荐从环境变量中获取。
*   打印函数返回结果，检查输出是否符合预期。
*   **注意**: 运行中可能会出现 `wikipedia` 库的 `warning` 警告，通常不影响功能。

#### **9. 关键概念回顾**

*   **LangChain Expression Language (LCEL)**：`提示模板 | 模型` 这种简洁的链式组装方式。
*   **`invoke()`**: 用于执行链或模型的关键方法，传入一个包含变量值的字典。
*   **`.content`**: 从LLM返回的“消息对象”中提取实际文本内容。

*   `def generate_script(...)`: `def`是Python中定义函数的关键字。函数是一种可重用的代码块，你给它一些输入（我们称之为“参数”），它会执行一些操作，然后返回一个结果。

### **3. 定义两个聊天提示词模板 (Chat Prompt Templates)**

这是定义了你将要发送给AI的“指令”（或者说“问题”）的结构。

```python
    title_template = ChatPromptTemplate.from_messages(
        [
            ("human", "请为'{subject}'这个主题的视频想一个吸引人的标题")
        ]
    )
    script_template = ChatPromptTemplate.from_messages(
        [
            ("human",
             """你是一位短视频频道的博主。根据以下标题和相关信息，为短视频频道写一个视频脚本。
             视频标题：{title}，视频时长：{duration}分钟，生成的脚本的长度尽量遵循视频时长的要求。
             要求开头抓住限球，中间提供干货内容，结尾有惊喜，脚本格式也请按照【开头、中间，结尾】分隔。
             整体内容的表达方式要尽量轻松有趣，吸引年轻人。
             脚本内容可以结合以下维基百科搜索出的信息，但仅作为参考，只结合相关的即可，对不相关的进行忽略：
             ```{wikipedia_search}```""")
        ]
    )
```

*   **`title_template` (标题模板)**:
    *   `ChatPromptTemplate.from_messages(...)`: 这表示我们正在创建一个聊天提示词模板。
    *   `[("human", "...") ]`: 这是一个列表，里面包含了一个元组。
        *   `"human"`：表示这条消息是由“人类”发出的（也就是我们的程序）。在与聊天AI对话时，通常需要区分是谁在说话（比如人类、AI助手、系统）。
        *   `"请为'{subject}'这个主题的视频想一个吸引人的标题"`：这是给AI的实际指令。
            *   `'{subject}'`: 这是一个占位符。当你实际使用这个模板时，这个`'{subject}'`会被你传入的真正主题替换掉（比如“Sora模型”）。

*   **`script_template` (脚本模板)**:
    *   这个模板更复杂，包含了更多的指令和占位符，因为它要生成整个视频脚本。
    *   再次出现`("human", "...")`，这仍然是给AI的指令。
    *   指令内容分解：
        *   `"你是一位短视频频道的博主。"`：这给AI设定了一个“角色”（persona）。告诉AI要以博主的身份来思考和写作，这有助于AI的语气和风格更符合目标。
        *   `"根据以下标题和相关信息，为短视频频道写一个视频脚本。"`：这是 AI 的主要任务。
        *   `"视频标题：{title}，视频时长：{duration}分钟，生成的脚本的长度尽量遵循视频时长的要求。"`：这里引入了两个新的占位符`{title}`和`{duration}`。AI会根据这两个信息来设计脚本。
        *   `"要求开头抓住限球，中间提供干货内容，结尾有惊喜，脚本格式也请按照【开头、中间，结尾】分隔。"`：这些是明确的结构和内容要求，指导AI如何组织脚本。
            *   `开头抓住限球`：通常指的是开头要吸引人，引人入胜。
            *   `中间提供干货内容`：指视频主体要有实质性的知识点或信息。
            *   `结尾有惊喜`：指结尾要有亮点或引发思考。
            *   `【开头、中间，结尾】分隔`：要求AI在生成脚本时，用这些词语把不同部分明确地分开，方便我们阅读。
        *   `"整体内容的表达方式要尽量轻松有趣，吸引年轻人。"`：这指定了脚本的风格和目标受众的语气。
        *   `"脚本内容可以结合以下维基百科搜索出的信息，但仅作为参考，只结合相关的即可，对不相关的进行忽略："`：这是一个非常关键的指令。
            *   它告诉AI，接下来会提供一些维基百科的搜索结果。
            *   `但仅作为参考`：说明AI不需要把所有信息都用上。
            *   `只结合相关的即可，对不相关的进行忽略`：要求AI进行信息筛选和判断，只使用与视频主题和脚本相关的部分。
            *   `` `{wikipedia_search}``` ``：这是另一个占位符，用来将维基百科的搜索结果插入到提示词中。注意这里的三个反引号（```）在Markdown或AI提示中常用于包裹代码块或大段文本，可以帮助AI更好地识别这部分内容是参考信息。

### **4. 设置AI模型 (Set up the AI Model)**

```python
    model = ChatOpenAI(openai_api_key=api_key, temperature=creativity)
```

*   `model = ChatOpenAI(...)`: 这一行就是根据你提供的参数，**创建一个可以和OpenAI聊天AI对话的“实例”**（你可以理解成在你的程序里实例化了一个AI模型对象）。
    *   `openai_api_key=api_key`: 这里把你在函数开头传入的`api_key`赋值给OpenAI模型的认证参数。这是告诉OpenAI你身份的凭证，就像一把钥匙，没有它你就无法使用OpenAI的服务。
    *   `temperature=creativity`: 这里把你在函数开头传入的`creativity`数值赋值给AI模型的`temperature`参数，控制AI生成内容的创意度。

### **5. 创建LangChain“链” (Create LangChain "Chains")**

```python
    title_chain = title_template | model
    script_chain = script_template | model
```

*   `|`：这个竖线符号在LangChain中被称为 **“管道（Pipe）”操作符**，或者说 **“链（Chain）”操作符**。它的意思是“把左边的输出作为右边的输入”。
    *   `title_chain = title_template | model`: 这行代码创建了一个名为`title_chain`的“工作流”。它表示：
        1.  首先使用`title_template`（标题模板）来构建完整的提示词。
        2.  然后把这个提示词的输出（完整的指令）传递给`model`（AI模型）。
        3.  `title_chain`就代表了这个从模板到AI模型，并最终得到AI标题回复的整个过程。
    *   `script_chain = script_template | model`: 同理，这创建了一个名为`script_chain`的工作流，用于生成脚本。它将`script_template`构建的提示词传递给`model`来生成视频脚本。

---

### **6. 生成视频标题 (Generate Video Title)**

```python
    title = title_chain.invoke({"subject": subject}).content
```

*   `title_chain.invoke(...)`: 这是真正 **“启动”** 或 **“运行”** `title_chain`这个工作流的命令。
    *   `{"subject": subject}`: 这是一个Python字典。在这里，我们把函数接收到的`subject`变量的值，赋值给`title_template`中定义的`{subject}`占位符。例如，如果`subject`是“Sora模型”，那么`title_template`就会变成“请为'Sora模型'这个主题的视频想一个吸引人的标题”。
*   `.content`: 当AI模型给出回复后，这个回复会是一个复杂的对象。我们通常只关心AI生成的实际文本内容，`.content`就是用来从AI的回复对象中提取出纯文本内容。
*   `title = ...`: AI生成的标题文本会被存储在`title`这个变量中。

---

### **7. 获取维基百科搜索结果 (Get Wikipedia Search Results)**

```python
    search = WikipediaAPIWrapper(lang="zh")
    search_result = search.run(subject)
```

*   `search = WikipediaAPIWrapper(lang="zh")`: 创建一个维基百科搜索工具的实例。`lang="zh"`表示我们希望搜索中文维基百科。
*   `search_result = search.run(subject)`: 
    *   `search.run(subject)`：调用`search`对象的`run`方法，传入`subject`（视频主题），让它去维基百科上搜索与主题相关的信息。
    *   搜索到的结果（可能是一长串文字）会被存储在 `search_result` 这个变量中。

---

### **8. 生成视频脚本 (Generate Video Script)**

```python
    script = script_chain.invoke({"title": title, "duration": video_length,
                                  "wikipedia_search": search_result}).content
```

*   `script_chain.invoke(...)`: 同样，这是 **“启动”** `script_chain`这个工作流的命令。
*   `{"title": title, "duration": video_length, "wikipedia_search": search_result}`: 这次我们传入了三个变量来填充`script_template`中的占位符：
    *   `"title": title`: 使用之前AI生成的标题。
    *   `"duration": video_length`: 使用函数接收到的视频时长。
    *   `"wikipedia_search": search_result`: 传入刚刚从维基百科搜索到的信息。
*   `.content`: 提取AI生成的完整脚本文本。
*   `script = ...`: AI生成的脚本文本会被存储在`script`这个变量中。

---

### **9. 返回结果 (Return Results)**

```python
    return search_result, title, script
```

*   `return ...`: `return`关键字表示函数执行完毕后，会将这些值作为结果返回给调用它的地方。
*   这里返回了三个值：
    *   `search_result`: 维基百科的原始搜索结果。
    *   `title`: AI生成的视频标题。
    *   `script`: AI生成的完整视频脚本。

### **总结一下**

这段代码是一个非常好的例子，展示了如何结合不同的工具和AI模型来完成一个复杂的任务：
1.  **AI模型 (ChatOpenAI)**：负责核心的标题和脚本生成。
2.  **提示词模板 (ChatPromptTemplate)**：以结构化的方式告诉AI要做什么，以及如何做。
3.  **工具 (WikipediaAPIWrapper)**：为AI提供了额外的背景信息，让AI能够创作出更丰富、更准确的内容。
4.  **LangChain (管道操作符 `|` 和 `invoke`)**：提供了一种流畅的方式来连接这些不同的组件，形成一个完整的工作流程。
