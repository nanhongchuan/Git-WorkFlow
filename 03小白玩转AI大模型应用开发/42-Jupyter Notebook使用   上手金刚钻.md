# 42.Jupyter Notebook使用   上手金刚钻

## 1. 启动与访问

- 启动方式
  - Windows：开始菜单搜索“cmd”，打开命令提示符
  - macOS：点击右上角放大镜，搜索“Terminal”，打开终端
  - 启动命令（回车后会自动打开浏览器）：
    ```bash
    jupyter notebook
    ```
- 找回页面地址
  - 如果误关了浏览器页面，可在启动命令的控制台/终端中找到包含“Jupyter Server is running at ...”的行，复制其后的地址到浏览器即可访问。
- 切记不要关闭启动它的控制台/终端窗口，否则 Notebook 服务会被终止。

## 2. 主面板与新建笔记本

- 主面板会展示“当前目录”的所有文件/文件夹，可点击进入不同目录。
- 新建 Notebook
  - 先进入你希望保存的位置（例如“桌面”），再点击“New Notebook”（或相应的新建入口）。
  - 会自动打开新的编辑页面，同时在该目录下生成对应文件。
- 重命名
  - 在编辑界面顶部点击标题，直接输入新名称即可。

## 3. 编辑界面构成

- 标题行：文件名（可直接修改）
- 菜单栏 + 工具条：工具条集成了最常用操作，优先通过工具条与单元格交互
- 单元格（Cell）：用于书写并执行代码

## 4. 单元格模式与切换

- 编辑模式：点击单元格进入，底色变白；可编辑内容
- 命令模式：按 Esc 或点击其他区域，底色变灰；用于对单元格进行操作（新增、删除、移动等）

## 5. 工具条常用按钮与操作

- 保存按钮：保存当前 Notebook 内容
- 新建单元格（在当前单元格下方）
- 剪切 / 复制 / 粘贴单元格
  - 可按住 Shift 选择多个单元格后批量操作
- 运行按钮（Run）
  - 执行当前单元格全部代码
  - 左侧方括号显示执行状态：
    - [*]：运行中
    - [数字]：执行完成后显示执行序号（如 [1]、[2]）
  - 可按任意顺序运行任意单元格，也可多次重复运行同一单元格
  - 执行序号用于“回溯/判断”运行顺序与状态

## 6. 执行顺序与变量状态（重要细节）

- 典型场景
  - 第1格定义变量，第2格打印变量
  - 若修改了变量，需要“先重新运行定义变量的单元格”，再运行打印的单元格
- 用执行序号判断是否最新
  - 如果第1格（定义变量）的序号 > 第2格（打印）的序号，说明打印结果还基于旧值，需要重新运行第2格
- 拆分单元格的意义
  - 将“耗时或计费”的步骤拆出，避免每次从头运行，节省时间与 API 费用
- 输出行为（交互式内核细节）
  - 单元格中无需显式 print，最后一个表达式的值会自动展示
  - **若同一单元格有多项输出，只有“最后一项”会显示；需使用 print 展示多项**
  - 示例：
    ```python
    a = 1
    a  # 这一行会显示 1
    b = 2
    b  # 只有最后一个表达式（b）的值会显示为 2

    # 展示多项输出需使用 print
    print(a)
    print(b)
    ```

## 7. 中断与重启（内核相关）

- 中断执行（Interrupt）按钮：正在运行代码时可中止当前执行
- 重启（Restart）按钮：
  - 清空所有已定义的变量与状态
  - **重启后，单元格序号会从 [1] 重新开始**
  - **例如：若只运行“打印变量”的单元格，会报“变量未定义”，需先运行“定义变量”的单元格**
- 重启并运行所有单元格（Restart & Run All）：
  - **从上到下完整执行一遍**
  - 用于验证 Notebook 的可复现性与单元格顺序是否正确

## 8. 使用建议与小贴士

- 开发节奏
  - 通过拆分单元格，实现最小化运行范围；针对修改点局部运行，提升效率
- 状态管理
  - 经常使用“Restart & Run All”检查依赖顺序、全局状态是否一致
- 界面操作
  - 重命名直接点标题即可；多选单元格用 Shift
- 稳定性
  - 不要关闭启动 Jupyter 的控制台/终端；如误关浏览器页面，可在终端日志中找回访问地址

## 9. 常用命令（速查）

```bash
# 启动 Jupyter Notebook
jupyter notebook
```
