# 115-部署应用 让你的AI应用能用链接访问

**使用 Streamlit Community Cloud 部署 AI 应用到可访问链接**

**一、 前提条件**
*   项目代码已上传到 GitHub 仓库。

**二、 登录 Streamlit Community Cloud**
1.  **访问链接：** `share.streamlit.io`
2.  **注册/登录方式：**
    *   可专门注册账号。
    *   推荐使用 **"Continue with GitHub"** (用 GitHub 账号登录)，方便快捷。
3.  **首次登录/授权：**
    *   可能会出现 GitHub 授权页面，点击绿色按钮授权 Streamlit 使用你的 GitHub 信息。
    *   可能需要填写个人信息（如名字、邮箱、国家）等必填项。

**三、 开始部署新应用**
1.  成功登录并进入主页面后，点击 **"New APP"** 按钮。
2.  **GitHub 仓库授权：** 第一次部署时，可能会再次出现授权页面，要求授权 Streamlit 访问你的 GitHub 仓库信息（点击绿色按钮授权）。授权一次后，后续部署不再需要。

**四、 填写部署项目信息**
1.  **Repository (仓库名):**
    *   点击下拉框，选择你需要部署的 GitHub 项目仓库。
2.  **Branch (分支):**
    *   对应要部署的分支。如果你不了解分支功能，通常无需修改，保持默认选项即可 (如 `main` 或 `master`)。
3.  **Main file path (主文件路径):**
    *   Streamlit 项目的网站主文件，即你在本地运行 `streamlit run <文件名>` 时输入的那个文件名。
    *   示例：`main.py`。
4.  **URL (应用链接):**
    *   系统会自动为你生成一个链接。
    *   可以自定义修改为更简短或易于记忆的名称。
    *   确保自定义名称下方显示 **绿色文字**，表示该 URL 未被占用。

**五、 启动部署与监控**
1.  填写完所有信息后，点击 **"Deploy"** 按钮开始部署应用。
2.  **部署过程：**
    *   页面上会显示加载小动画。
    *   若想查看部署流程，可以点击下方的 **"Manage APP"** 按钮，它会展示后台实际运行的命令，包括安装 `requirements.txt` 中所有依赖项的过程。

**六、 部署完成与分享**
1.  通常等待几分钟后，应用即可部署成功。
2.  部署完成后，你的 AI 应用将通过你设置的 URL 链接对外公开访问。
3.  你可以将此链接发送给他人，让他们使用你构建的应用。
