# 🧭 内网服务器如何设置代理访问外网（完整教程）

适用场景：服务器 A（内网机器）没有外网权限，需要通过服务器 B（有公网权限）访问互联网。

参考文章：https://cloud.tencent.com/developer/article/1748416

---

## 🗂 环境说明

| 服务器 | IP             | 系统     | 特点             |
|--------|----------------|----------|------------------|
| A      | 172.19.58.202  | CentOS 7 | 无外网权限        |
| B      | 172.19.58.201  | CentOS 7 | 有公网出口（代理） |

---

## ✅ 方法一：使用 Squid 正向代理

### 📌 步骤一：在 B 上安装并配置 Squid

```bash
yum install -y squid
```

编辑配置文件 `/etc/squid/squid.conf`：

```bash
# 注释默认 deny 规则
# http_access deny all

# 开放访问权限
http_access allow all

# 修改监听端口
http_port 10991
```

初始化并启动服务：

```bash
squid -k parse       # 检查配置是否正确
squid -z             # 初始化缓存目录
service squid start  # 启动 squid 服务
netstat -an | grep 10991  # 查看端口是否监听
```

### 📌 步骤二：在 A 设置环境变量代理

编辑环境变量配置文件：

```bash
vi /etc/profile.d/proxy.sh
```

添加如下内容：

```bash
export http_proxy=http://172.19.58.201:10991
export https_proxy=http://172.19.58.201:10991
export no_proxy="127.0.0.1,localhost,172.19.58.202,172.19.58.201"
```

保存并使其生效：

```bash
source /etc/profile.d/proxy.sh
echo $http_proxy  # 检查变量是否生效
```

测试访问外网是否成功：

```bash
curl https://www.google.com
```

## 🚀 方法二：使用 Nginx 正向代理

### 📌 步骤一：在 B 安装并配置 Nginx

```bash
yum install -y nginx
```

编辑配置文件 `/etc/nginx/nginx.conf`：

在 `http` 块中添加以下内容：

```nginx
server {
    listen 10991;

    resolver 8.8.8.8;

    location / {
        proxy_pass $scheme://$http_host$request_uri;

        proxy_set_header Host $http_host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;

        proxy_buffers 256 4k;
        proxy_max_temp_file_size 0;
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }
}
```

保存后重启 Nginx：

```bash
nginx -t  # 检查配置
systemctl restart nginx
netstat -anp | grep 10991  # 查看监听端口
```

### 📌 步骤二：在 A 配置代理访问

与 Squid 配置相同：

```bash
vi /etc/profile.d/proxy.sh
```

```bash
export http_proxy=http://172.19.58.201:10991
export https_proxy=http://172.19.58.201:10991
export no_proxy="127.0.0.1,localhost,172.19.58.202,172.19.58.201"
```

生效并测试：

```bash
source /etc/profile.d/proxy.sh
curl https://www.google.com
```

---

## 📌 补充说明

- 如果 A 上 curl 访问失败，请检查：
  - B 的防火墙是否放行 10991 端口；
  - squid/nginx 服务是否正常启动；
  - `/etc/resolv.conf` 是否配置了可用 DNS（如 8.8.8.8）；
  - `iptables` 或 `firewalld` 规则是否允许 A 访问 B。

## 📚 总结对比

| 项目        | Squid                         | Nginx                          |
|-------------|-------------------------------|--------------------------------|
| 类型        | 专业 HTTP 代理服务            | 通用 Web 服务器+代理能力      |
| 配置复杂度  | 略高                          | 简单                          |
| 功能        | 代理协议更丰富（支持 FTP 等） | 适用于 HTTP/HTTPS 简单代理    |
| 适合场景    | 高并发代理、大型部署          | 小规模快速配置                |
