# 72-爆款小红书文案生成器 创建AI请求

**一、本节核心任务：**
*   完成对AI的请求，将小红书主题传递给AI。
*   获取AI返回的五个待选标题和生成的正文。

**二、项目初始化与依赖安装：**
1.  **创建项目文件夹。**
2.  **下载 `requirements.txt` 文件：** 拖入项目文件夹。
3.  **安装依赖：** 在终端运行 `pip install -r requirements.txt`。
    *   **关键依赖：** `streamlit` (前端), `openai` (AI模型), `langchain` (AI应用框架), `pydantic` (数据校验)。

**三、AI交互代码文件 (`ai.py`) 创建与函数定义：**
1.  **新建文件：** `ai.py` (存放所有与AI大模型交互的代码)。
2.  **定义主函数 `generate_xiaohongshu`：**
    *   **参数：** `theme` (小红书主题), `api_key` (用户提供的API密钥)。
    *   **功能：** 串联“提示模板 -> 模型 -> 输出解析器”的完整链路。
    *   **关键环节：** 正确解析AI返回的五个标题和一个正文，此处推荐使用 `langchain` 的 JSON 数据解析器。

**四、提示模板 (Prompt Template) 定义：**
1.  **新建文件：** `prompt_template.py` (专门存放模板文本，避免与代码混淆)。
2.  **内容：** 包含系统提示和用户提示。
    *   **注意：** 与之前的API示例有区别，不再指定输出格式，而是通过插入 `parser_instructions` 变量，让输出解析器提供格式指令。
3.  **导入：** 在 `ai.py` 中导入 `system_template_text` 和 `user_template_text`。
4.  **构建消息列表：**
    *   从 `langchain.prompts` 导入 `ChatPromptTemplate`。
    *   使用 `ChatPromptTemplate.from_messages` 方法构建 `prompt` 变量。
        *   `system` 角色对应 `system_template_text`。
        *   `user` 角色对应 `user_template_text`。

**五、聊天模型 (Chat Model) 定义：**
1.  **导入：** 从 `langchain.chat_models` 导入 `ChatOpenAI`。
2.  **创建模型实例：**
    *   `ChatOpenAI(api_key=api_key)`：将用户提供的 `api_key` 赋值给模型。
    *   **可选参数：** 如果使用课程提供的API密钥进行测试，需额外提供 `openai_api_base` 参数。

**六、输出解析器 (Output Parser) 定义：**
1.  **选择解析器：** `PydanticOutputParser` 是理想选择，可解析 JSON 格式输出到自定义类实例，并强制类型检查。
    *   从 `langchain.output_parsers` 导入 `PydanticOutputParser`。
2.  **定义数据模型 (`xiaohongshu_model.py`)：**
    *   **新建文件：** `xiaohongshu_model.py`。
    *   **定义类：** `Xiaohongshu`，继承自 `BaseModel` (从 `pydantic` 导入)。
    *   **属性定义：**
        *   `titles`: `List[str]` (包含五个标题的列表)。
            *   使用 `typing.List` 导入。
            *   使用 `Field` 函数 (`pydantic.Field`) 添加描述 (`小红书的五个标题`) 和校验 (`min_items=5`, `max_items=5`)。
        *   `content`: `str` (小红书正文内容)。
            *   使用 `Field` 函数添加描述 (`小红书的正文内容`)。
3.  **创建解析器实例：** 在 `ai.py` 中导入 `Xiaohongshu` 类，并赋值给 `PydanticOutputParser` 的 `pydantic_object` 参数。

**七、串联组件与执行链 (Chain)：**
1.  **构建链：** 将 `prompt`、`model` 和 `output_parser` 串联起来。
2.  **调用链：**
    *   使用 `.invoke()` 方法触发链。
    *   **参数：** 字典形式的提示模板变量键值对。
        *   `parser_instructions`: `output_parser.get_format_instructions()` (提供AI输出格式指令)。
        *   `theme`: `theme` (函数接收的小红书主题参数)。
3.  **返回结果：** `result` 变量，其类型为 `Xiaohongshu` 类实例。

**八、测试与验证：**
1.  **调用 `generate_xiaohongshu` 函数：**
    *   提供测试主题 (任意)。
    *   提供API密钥 (可从环境变量获取 `os.environ.get("OPENAI_API_KEY")`)。
2.  **打印结果：** 验证返回的 `Xiaohongshu` 实例是否包含：
    *   `titles` 属性：一个包含五个字符串的列表。
    *   `content` 属性：一个字符串。
3.  验证结果格式是否符合预期，确认后端逻辑已成功实现。

# `os.getenv("OPENAI_API_KEY")` 是什么？

我们来一步一步看：

#### 1. `os` 是什么？

*   `os` 是 Python 标准库中一个非常重要的**模块**（module）。
*   你可以把模块想象成一个“工具箱”，里面装满了各种与操作系统（Operating System，比如 Windows, macOS, Linux）交互的工具。
*   这个 `os` 模块提供了很多方法（也就是工具箱里的工具），让你能够：
    *   处理文件和目录（比如创建文件夹、删除文件）。
    *   获取当前操作系统的信息。
    *   **处理环境变量（我们接下来要讲的重点）**。

*   **在使用 `os` 模块之前，通常需要先导入它：**
    ```python
    import os
    ```
    这条语句就像是把 `os` 这个工具箱搬到你的工作台前，这样你才能使用里面的工具。

#### 2. `getenv()` 是什么？

*   `getenv` 是 `os` 模块里面的一个**方法**（method），也可以叫做函数。
*   它的全称是 "get environment"（获取环境变量）。
*   它的作用就是：从当前计算机的**环境变量**中，查找并获取指定名称的值。

#### 3. `("OPENAI_API_KEY")` 是什么？

*   `"OPENAI_API_KEY"` 是你希望 `getenv()` 方法去查找的那个**环境变量的名称**。
*   它是一个字符串，表示环境变量的**键**（key）。当你调用 `os.getenv("某个名字")` 时，Python 就会去你的操作系统查找有没有一个叫做 "某个名字" 的环境变量，如果有，就返回它的值；如果没有，就返回 `None` (表示没找到)。

### 4. 什么是“环境变量”（Environment Variable）？

这是理解 `os.getenv()` 的核心概念。

想象一下：你的电脑上有一个**全局的“小黑板”**，上面可以写一些每个人（或每个程序）都可以来查看的信息。这些信息就是“环境变量”。

*   **名称-值对：** 每个环境变量都像一个“键-值”对，比如：
    *   `PATH = /usr/local/bin:/usr/bin` (这是系统用来找程序的路径)
    *   `HOME = /Users/youruser` (这是你的用户主目录)
    *   `OPENAI_API_KEY = sk-xxxxxxxxxxxxxxxxxxxxxxxx` (这就是你用来存储 OpenAI API 密钥的地方)

*   **全局可用：** 一旦你在操作系统中设置了一个环境变量，你的计算机上运行的各种程序（包括你的Python代码），都可以在不需要硬编码（直接写在代码里）的情况下，来读取这些值。

### 5. 为什么我们用 `os.getenv()` 来获取 `OPENAI_API_KEY`？

这是一个非常重要的安全和最佳实践问题：

*   **避免硬编码敏感信息：** API 密钥（比如 OpenAI 的密钥）是非常敏感的！如果 GITHUB 仓库代码里直接写上 `openai_api_key = "sk-xxxxxxxxxxxxxxxxxxxxxxxx"` ，然后你把代码上传到 GitHub，那么你的密钥就泄露了。任何看到你代码的人都能用你的密钥，可能导致你的账户被盗用，或者产生大量的费用！
*   **灵活性：** 如果你的密钥变了，或者你在不同的环境（比如开发环境、测试环境、生产环境）使用不同的密钥，你只需要修改操作系统中的环境变量，而不需要去修改 Python 代码。
*   **隔离：** 你的代码里不需要知道密钥的具体值，它只需要知道去哪里取这个值（即环境变量的名称）。

所以，`os.getenv("OPENAI_API_KEY")` 的作用就是：
**安全地从你的操作系统中，读取名为 `OPENAI_API_KEY` 的值，而这个值就是你的 OpenAI API 密钥。**

### 6. 如何设置环境变量？

要让 `os.getenv("OPENAI_API_KEY")` 能够取到值，你需要先在你的操作系统中设置这个环境变量。

**a) 临时设置（只在当前终端会话有效）：**

*   **在 Linux/macOS 终端：**
    ```bash
    export OPENAI_API_KEY="sk-你的真实API密钥"
    ```
    （注意：`=` 两边不能有空格）
    执行这行命令后，你再运行你的 Python 脚本，它就能取到这个值了。但你关掉终端窗口后，这个设置就失效了。

*   **在 Windows 命令提示符 (cmd)：**
    ```cmd
    set OPENAI_API_KEY="sk-你的真实API密钥"
    ```
*   **在 Windows PowerShell：**
    ```powershell
    $env:OPENAI_API_KEY="sk-你的真实API密钥"
    ```

**b) 永久设置（推荐用于开发环境）：**

*   **在 Linux/macOS：**
    通常编辑 `~/.bashrc` 或 `~/.zshrc`（取决于你使用的 Shell），在文件末尾添加 `export OPENAI_API_KEY="sk-你的真实API密钥"`，然后保存文件并运行 `source ~/.zshrc` (或 `source ~/.bashrc`) 来更新。
*   **在 Windows：**
    通过“系统属性” -> “高级” -> “环境变量” 手动添加或修改。或者使用第三方工具如 `python-dotenv` 库。

### 总结一下 `os.getenv("OPENAI_API_KEY")` 的完整流程：

1.  你的 Python 脚本执行 `import os`，加载 `os` 模块。
2.  脚本调用 `os.getenv("OPENAI_API_KEY")`。
3.  `os` 模块向操作系统查询是否存在一个名为 `OPENAI_API_KEY` 的环境变量。
4.  如果操作系统找到了这个环境变量，它就把对应的值（比如 `sk-xxxxxxxxxxxxxxxxxxxxxxxx`）返回给 `os.getenv()`。
5.  `os.getenv()` 再将这个值返回给你的 Python 代码，作为 `generate_xiaohongshu` 函数的第二个参数。
6.  `generate_xiaohongshu` 函数内部就可以使用这个密钥来调用 OpenAI API 了。

所以，`os.getenv("OPENAI_API_KEY")` 的作用就是**安全、灵活地获取你的 OpenAI API 密钥**。
