**II. 对话记忆与会话状态管理 (核心)**

1.  **记忆初始化 (ConversationBufferMemory)**
    *   **目的：** 存储历史对话，使AI能进行连贯对话。
    *   **导入：** 从 `memory` 模块导入 `ConversationBufferMemory`。
    *   **创建实例：** `ConversationBufferMemory(return_messages=True)`
        *   `return_messages=True`：确保存储的是消息列表，而非一整段字符串。
    *   **问题：** Streamlit 会在用户交互或代码修改时重新运行全部代码，导致每次交互后记忆被重新初始化，历史对话被清空。
    *   **解决方案：使用 `st.session_state` (会话状态)**
        *   **机制：** 只有当 `memory` 键不在 `st.session_state` 中时，才进行初始化。
        *   **代码逻辑：**
            ```python
            if "memory" not in st.session_state:
                st.session_state.memory = ConversationBufferMemory(return_messages=True)
            ```

2.  **消息列表初始化 (st.session_state.messages)**
    *   **目的：** 在前端页面上展示对话内容。
    *   **存储位置：** 使用 `st.session_state` 存储一个消息列表。
    *   **代码逻辑：**
        ```python
        if "messages" not in st.session_state:
            st.session_state.messages = []
            # 添加一条初始的AI消息
            st.session_state.messages.append(
                {"role": "AI", "content": "你好，我是你的AI助手，有什么可以帮你的吗？"}
            )
        ```
    *   **消息元素格式：** 每个元素是一个字典，包含 `role` (角色) 和 `content` (实际消息内容)。
        *   `role`：用于区分消息来源，"AI" 表示AI，"human" 表示人类用户。

**III. 展示历史对话**

1.  **遍历消息列表**
    *   使用 `for` 循环迭代 `st.session_state.messages` 中的每条消息。
2.  **使用 `st.chat_message()` 函数**
    *   **用法：** `st.chat_message(role_string).write(content_string)`
        *   `role_string`：消息来源的角色字符串（如 "AI" 或 "human"）。
        *   `content_string`：消息的实际内容。
    *   **示例：**
        ```python
        for message in st.session_state.messages:
            st.chat_message(message["role"]).write(message["content"])
        ```
    *   Streamlit 会根据 `role` 自动选择图标来表示角色。

**IV. 用户输入与AI回应逻辑**

1.  **用户消息输入框**

2.  **API密钥检查**
    *   在处理用户消息之前，检查用户是否提供了API密钥 (`st.session_state.api_key`)。
    *   如果未提供，显示警告信息 (`st.warning("请输入你的OpenAI API密钥")`)，并停止后续代码执行 (`st.stop()`)。
3.  **处理用户输入**
    *   **存储：** 将用户输入的消息添加 (`append`) 到 `st.session_state.messages` 列表中。
        *   格式：`{"role": "human", "content": user_message}`
    *   **展示：** 使用 `st.chat_message("human").write(user_message)` 即时显示用户输入。
4.  **获取AI回应**
    *   **加载提示：** 使用 `with st.spinner("正在加载中..."):` 组件，告知用户需要等待。
    *   **调用 `get_chat_response`：**
        *   参数：用户消息，`st.session_state.memory`，API密钥（从会话状态中获取）。
        *   结果存储在 `response` 变量中。
5.  **处理AI回应**
    *   **存储：** 将AI回应构建成字典 `{"role": "AI", "content": response}`，并添加到 `st.session_state.messages` 列表中。
    *   **展示：** 使用 `st.chat_message("AI").write(response)` 将AI生成的消息内容显示在网页上。

    *   这个 `if` 语句检查 `st.session_state` 中是否已经存在一个名为 "memory" 的键。如果不存在，表示这是应用首次运行或用户开始了新的对话会话，需要进行初始化。

    *   `return_messages=True`：这个参数告诉 `ConversationBufferMemory` 在检索内存内容时，以 LangChain `Message` 对象的列表形式返回，而不是简单的字符串。这对于与 LLM 的高级交互很有用。

3.  **`st.session_state["messages"] = [{"role": "ai", "content": "你好，我是你的AI助手，有什么可以帮你的吗？"}]`**:

```python
prompt = st.chat_input()
if prompt:
    if not openai_api_key:
        st.info("请输入你的OpenAI API Key")
        st.stop()
```
