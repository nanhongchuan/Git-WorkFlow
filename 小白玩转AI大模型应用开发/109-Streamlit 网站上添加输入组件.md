# 109-Streamlit 网站上添加输入组件

#### 1. 概述

*   Streamlit不仅能展示内容，还能通过各种输入组件接收用户输入，实现动态交互。
*   本指南将介绍如何添加文本输入、数字输入、勾选框和按钮等常见组件。

#### 2. 项目设置与运行

*   **创建文件:** 在现有Streamlit项目或新项目中创建 `.py` 脚本文件 (e.g., `app.py`)。
*   **导入库:** 在脚本开头导入 `streamlit` 库，通常使用别名 `st`：
    ```python
    import streamlit as st
    ```
*   **运行应用:** 在终端中使用 `streamlit run` 命令启动应用：
    ```bash
    streamlit run your_app_name.py
    ```
    (注意替换为你的文件名)
*   **页面初始状态:** 当文件为空时，页面上显示为空白。

#### 3. Streamlit 页面运行机制 (核心概念)

*   **程序重新运行的两种情况:**
    1.  **源代码修改:** 当`.py`文件被修改并保存时，Streamlit会自动从头到尾重新运行整个脚本。
    2.  **用户交互:** 当用户与Streamlit页面上的任何组件发生交互时 (如：输入文本后回车、点击按钮、拖动滑块等)，整个脚本也会从头到尾重新运行。
*   **效果:** 每次重新运行时，脚本中的所有代码都会被执行，组件的当前值会更新相应的Python变量，进而更新页面显示。

#### 4. 核心输入组件详解

---

##### 4.1. 文本输入框 (`st.text_input`)

*   **用途:** 接收用户单行短文本输入 (如姓名)。
*   **函数:** `st.text_input(label)`
*   **参数:**
    *   `label` (必填, 字符串): 显示在输入框上方的提示文字，向用户说明要输入的内容。
*   **返回值:** 用户输入的字符串。
    *   当用户还未输入时，对应变量的值通常为 `None` 或空字符串。
*   **示例:**
    ```python
    name = st.text_input("请输入您的名字")
    if name: # 判断用户是否已输入
        st.write(f"你好，{name}！")
    ```
*   **交互:** 用户输入文本并回车后，会触发脚本重新运行，`name` 变量的值会被更新。

---

##### 4.2. 密码输入框 (`st.text_input` with `type='password'`)

*   **用途:** 接收密码等敏感信息，输入内容默认不可见。
*   **函数:** `st.text_input(label, type='password')`
*   **参数:**
    *   `type='password'` (字符串): 将输入框类型设置为密码模式。
*   **特点:**
    *   用户输入的内容会以点或星号显示，隐藏实际字符。
    *   输入框旁边通常有一个“小眼睛”图标，点击可切换显示/隐藏密码。
*   **示例:**
    ```python
    password = st.text_input("请输入密码", type='password')
    # ... 对密码进行处理
    ```

---

##### 4.3. 长文本区域 (`st.text_area`)

*   **用途:** 接收用户多行长文本输入 (如留言、评论、描述)。
*   **函数:** `st.text_area(label)`
*   **参数:**
    *   `label` (必填, 字符串): 提示文字，说明要输入的内容。
*   **特点:** 输入框通常可以手动拖动调整大小，以适应用户输入的文本长度。
*   **示例:**
    ```python
    comment = st.text_area("请留下您的留言")
    if comment:
        st.write("您的留言是:", comment)
    ```

---

##### 4.4. 数字输入框 (`st.number_input`)

*   **用途:** 接收用户数字输入 (如年龄、数量、分数)。
*   **函数:** `st.number_input(label, value=None, min_value=None, max_value=None, step=None, format=None)`
*   **参数:**
    *   `label` (必填, 字符串): 提示文字。
    *   `value` (可选): 输入框的初始默认值 (默认 `0.0`，精度2位小数)。
    *   `min_value` (可选): 允许输入的最小值。
    *   `max_value` (可选): 允许输入的最大值。
    *   `step` (可选): 点击输入框旁的加号或减号按钮时，数字每次增加或减少的量 (默认 `1`)。
*   **特点:**
    *   自动提供加 (`+`) 减 (`-`) 按钮，方便用户调整数字。
    *   如果用户输入的数字超出 `min_value` 或 `max_value` 范围，会自动显示错误提示。
*   **合理设置:** 根据应用场景，调整 `value`, `min_value`, `max_value`, `step` 参数以限制用户输入，提高数据合理性。
*   **示例 (年龄输入):**
    ```python
    age = st.number_input("请输入年龄", value=20, min_value=0, max_value=120, step=1)
    st.write(f"您的年龄是: {age} 岁")
    ```

---

##### 4.5. 勾选框 (`st.checkbox`)

*   **用途:** 接收用户是/否、同意/不同意等布尔值选择。
*   **函数:** `st.checkbox(label, value=False)`
*   **参数:**
    *   `label` (必填, 字符串): 显示在勾选框旁边的内容。
    *   `value` (可选, 布尔值): 勾选框的初始状态 (默认 `False`，即未勾选)。
*   **返回值:** 布尔值。
    *   `True`: 用户已勾选。
    *   `False`: 用户未勾选。
*   **常见用法:** 结合 `if` 语句，根据用户的勾选状态执行不同操作或显示不同内容。
*   **示例:**
    ```python
    agreed = st.checkbox("我已阅读并同意以上条款")
    if agreed:
        st.success("您已同意，可以继续操作。")
    else:
        st.info("请先勾选同意条款。")
    ```

---

##### 4.6. 按钮 (`st.button`)

*   **用途:** 触发特定动作或流程。
*   **函数:** `st.button(label)`
*   **参数:**
    *   `label` (必填, 字符串): 显示在按钮上的文字。
*   **返回值:** 布尔值。
    *   `True`: 用户刚刚点击了按钮。
    *   `False`: 按钮未被点击 (或上次点击后程序已重新运行)。
*   **重要特性:** 按钮被点击时，其返回值瞬间变为 `True`，并立即触发脚本重新运行，然后在下一次运行中，其值又变回 `False`。因此，`if st.button(...)` 语句只会在点击瞬间执行一次。
*   **常见用法:** 结合 `if` 语句，在按钮被点击时执行特定的代码块。
*   **示例:**
    ```python
    if st.button("提交表单"):
        st.success("表单已成功提交！")
        # 可以在这里添加提交数据的逻辑
    ```

---

#### 5. 总结

*   Streamlit提供了一系列简单易用的输入组件，让开发者能够快速构建具有用户交互功能的Web应用。
*   理解Streamlit的“从上到下重新运行”机制对于正确处理用户输入和更新页面至关重要。
