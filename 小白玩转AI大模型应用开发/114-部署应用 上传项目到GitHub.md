### 114-部署应用 上传项目到GitHub

#### **I. 应用部署概述**

*   **目的：** 让AI网站应用能被他人随时随地通过互联网访问。
*   **原理：** 从本地运行 (localhost) 部署到具有公网IP地址的服务器。
*   **Streamlit Cloud优势：** 提供便捷的部署流程，多数应用几分钟即可完成。

#### **II. 部署前准备：创建 `requirements.txt`**

*   **作用：** 记录项目所有依赖库的名称，供服务器识别和安装。
*   **创建方法：**
    1.  打开项目编辑器终端。
    2.  输入命令：`pip freeze > requirements.txt` (注意 `<` 及 `>` 符号前后各有一个空格)。
*   **结果：** 项目文件夹下会生成一个名为 `requirements.txt` 的文件，包含项目直接及间接依赖的所有库（你安装的库还依赖于别的库）。

#### **III. 上传项目到 GitHub**

*   **目的：** Streamlit社区云服务将从GitHub仓库拉取您的代码进行部署。
*   **GitHub网址：** `github.com`
*   **GitHub简介：** 全球最大的代码托管网站和开源社区，用于管理代码、追踪历史记录、查看他人项目。

---

**A. 注册/登录 GitHub 账号**

*   **注册流程 (Sign up)：**
    1.  输入邮箱 (`email`)。
    2.  设置密码 (`password`)：要求 **8位及以上数字+字母**，或 **15位及以上**。
    3.  设置用户名 (`username`)：只能包含**字母、数字、短横杠** (横线 `-`)，**不支持中文字符**。
    4.  是否接收宣传邮件：可选。
    5.  完成机器人验证。
    6.  输入邮箱收到的验证码。
*   **登录 (Sign in)：** 若已有账号。

---

**B. 创建新的代码仓库 (Repository)**

*   **作用：** 所有文件都需存放在特定的代码仓库下 (可理解为项目文件夹)。
*   **步骤：** 登录后点击 `New`。
*   **配置项：**
    1.  **Repository name (仓库名称)：** 只能包含字母、数字、英文句号、下划线、短横杠。
    2.  **Description (描述)：** 简短说明仓库内容，可使用中文。
    3.  **Public/Private (可见性)：**
        *   `Public`：他人可见。
        *   `Private`：仅自己和指定协作者可见。
        *   **建议：** 若需分享给他人看，选择 `Public`。
    4.  **Add a README file (添加README文件)：** 可选，用于提供项目介绍 (`.md` 后缀，Markdown格式)。
    5.  `.gitignore`：可先忽略。
    6.  **License (软件许可证)：** 规定和限制软件用户使用代码的权利。(如：署名要求、不可商用等)。
        *   了解不同许可证：https://choosealicense.com/
*   **完成：** 点击 `Create repository`。

---

**C. 上传文件到仓库**

*   **步骤：** 进入新创建的仓库 -> 点击 `Add file` -> `Upload files`。
*   **操作：**
    1.  将项目所有相关文件和文件夹拖拽至上传区域。
    2.  **重要排除项：`venv` 文件夹** (虚拟环境)。
        *   **原因：** 体积巨大，包含Python解释器和库，并非项目源代码，无需上传。
    3.  **Commit changes (提交说明)：**
        *   第一个文本框：简短描述本次代码提交的目的 (必填)。
        *   第二个文本框：提供更多详细信息 (选填)。
*   **完成：** 点击绿色的 `Commit changes` 按钮。

---

**D. 仓库文件管理**

*   **查看文件内容：** 点击文件名即可进入查看。
*   **删除文件：**
    1.  进入文件详情页。
    2.  点击右上角 `...` (三个点) 按钮。
    3.  选择 `Delete file`。
    4.  再次点击 `Commit changes` 确认提交。
*   **更新文件：**
    1.  直接上传具有相同文件名的文件。
    2.  旧内容将自动被新上传的文件覆盖。

---

**E. (可选了解) Git 版本控制**

*   **本质：** GitHub底层基于Git版本控制系统。
*   **强大功能：** 记录项目每次改动、改动者，可随时切换到之前版本状态。
*   **应用场景：** 在团队协作中特别有用。
*   **建议：** 感兴趣可深入学习Git。
