# 67-视频脚本一键生成器 项目介绍

**项目名称：** 视频脚本一键生成器

**一、项目目标与实现效果**

*   **核心功能：** 根据用户输入的主题和时长，结合AI大模型自动生成短视频脚本。
*   **用户体验：** 提供简洁的用户界面，方便用户快速操作。

**二、关键功能模块**

1.  **侧边栏 (Sidebar)：**
    *   **OpenAI API 密钥输入：**
        *   允许用户提供自己的 OpenAI API 密钥，以便使用用户自己的 token 额度。
        *   附带官方网站链接，引导用户获取密钥。
    *   **可拓展性：**
        *   可替换为其他大模型（如文心一言）的密钥输入。
        *   针对自用网站，可移除输入框，直接在代码中使用预设密钥。

2.  **主页面 (Main Page)：**
    *   **用户输入：**
        *   **视频主题：** 任意文本输入。
        *   **脚本大致时长：** 任意数字输入。
    *   **AI 创造力控制 (Temperature 参数)：**
        *   提供滑动条或输入框，范围 0-1。
        *   **对应 API 的 `temperature` 参数：**
            *   `temperature` 越低 (接近0)，AI 回答的随机性、创造性越低，结果更确定、保守。
            *   `temperature` 越高 (接近1)，AI 回答的随机性、创造性越高，可能收到出乎意料或更具创意的回答。

3.  **脚本生成流程：**
    *   **触发：** 用户点击“生成脚本”按钮。
    *   **前置检查：** 网站判断用户是否已提供 OpenAI API 密钥。
    *   **加载提示：** AI 模型生成回复需要时间，使用加载组件（如动效）告知用户“AI 还在思考中”。

4.  **结果展示：**
    *   显示“视频脚本已生成”的提示。
    *   展示生成的视频标题。
    *   展示生成的脚本内容。

**三、核心技术细节 (结合AI与外部信息)**

*   **克服模型知识局限性：** 解决大模型训练日期限制导致知识滞后或缺失的问题。
*   **方法：调用外部 API 获取实时/真实信息**
    *   **具体实践：** 在向 AI 模型发送请求之前，先调用 **维基百科 API**。
    *   **信息整合：** 将维基百科搜索出的内容，与用户要求（主题、时长、创造力）**联合传给 AI 模型**。
    *   **目的：**
        *   保证 AI 拥有**真实且实时**的补充信息。
        *   避免 AI 在缺乏信息时“胡编乱造”。

**四、项目意义与价值**

*   提供一个实用工具，简化短视频内容创作流程。
*   展示了如何结合大模型（LLM）与外部数据源（如维基百科）来增强 LLM 的能力和生成内容的准确性。
