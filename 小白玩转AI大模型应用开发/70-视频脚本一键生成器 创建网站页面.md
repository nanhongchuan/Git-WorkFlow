# 70-视频脚本一键生成器 创建网站页面

**一、项目背景与目标**

*   **问题痛点：** 上一节创建的 `generate_script` 函数虽然能生成视频主题、标题和脚本，但仅限代码调用者使用，不用户友好。
*   **解决方案：** 围绕此核心功能构建一个Web网站，使其面向所有用户，可随时随地使用。

**二、前端技术选型：Streamlit**

*   **选择原因：**
    *   Python库，无需学习其他前端语言。
    *   新手友好，上手快。
*   **学习资源：** 若不熟悉Streamlit，可查看课程结尾的加餐部分。
*   **替代方案：** 若已掌握其他前端框架，可自由选择。

**三、项目初始化与基本结构**

1.  **代码清理：** 删除或注释掉上一节中测试 `generate_script` 的代码。
2.  **新建文件：** 在项目文件夹中新建一个Python文件（例如 `app.py`），作为网站的主页。
3.  **导入模块：**
    *   `import streamlit as st` (核心Streamlit库)
    *   `from your_module import generate_script` (导入上一节创建的核心函数，假设在 `your_module.py` 中)
    *   注意：此文件无需导入Lanchain相关库，因为 `generate_script` 已封装。
4.  **运行命令：** `streamlit run your_script_name.py` (在终端中运行您的网站文件)。
5.  **实时更新：** 保存代码后，在网页上点击 "Always rerun" 或按 `R` 键，改动会在网页上实时生效。

**四、Streamlit组件及页面布局**

1.  **页面标题：** `st.title("视频脚本生成器")`
2.  **侧边栏 (Sidebar)：**
    *   使用 `with st.sidebar:` 结构，其内部所有缩进内容都会出现在侧边栏。
    *   **API密钥输入框：**
        *   `openai_api_key = st.sidebar.text_input("请输入OpenAI API密钥:", type="password")`
        *   `type="password"`：隐藏用户输入的密钥内容，提高安全性。
        *   返回值：用户输入的字符串。
    *   **获取密钥链接：**
        *   `st.sidebar.markdown("获取OpenAI API密钥 [<sup>1</sup>](https://platform.openai.com/api-keys)")`
        *   `st.markdown()`：用于在网页上添加Markdown格式的内容（如链接）。
3.  **主页面组件：**
    *   **视频主题输入框：** `subject = st.text_input("请输入视频的主题")` (注意：无缩进，不在侧边栏)。
    *   **视频时长输入框 (数字)：**
        *   `video_length = st.number_input("请输入视频的大致时长 (分钟):", min_value=0.1, step=0.1)`
        *   `min_value`：最小值限制。
        *   `step`：每次点击加减按钮的变化幅度。
        *   返回值：默认是 `float` 类型，无需手动转换。
    *   **AI创造力调节器 (滑块)：**
        *   `creativity = st.slider("调整AI回答的创造力 (温度):", min_value=0.0, max_value=1.0, value=0.2, step=0.1)`
        *   `min_value`：最小值。
        *   `max_value`：最大值（建议不要超过1.0，避免AI输出混乱）。
        *   `value`：默认值。
        *   `step`：每次拖动滑块的变化幅度。
        *   注意：`min_value` 和 `max_value` 的类型应与 `step` 保持一致（float类型）。
    *   **提交按钮：**
        *   `if st.button("生成脚本"):`
        *   `st.button()`：返回一个布尔值 ( `True` 为点击， `False` 为未点击)。
        *   所有的逻辑判断和生成请求都应放在这个条件判断块内。

**五、核心逻辑与用户交互 (按钮点击后)**

1.  **条件判断：** 所有请求逻辑放在 `if st.button("生成脚本"):` 块内。
2.  **输入校验 (顺序重要)：**
    *   **Step 1: API密钥校验**
        *   `if not openai_api_key:`
        *   `st.info("请输入您的OpenAI API密钥。")` (显示提示信息)
        *   `st.stop()` (停止执行后续代码)
    *   **Step 2: 视频主题校验**
        *   `if not subject:`
        *   `st.info("请输入视频的主题。")`
        *   `st.stop()`
    *   **Step 3: 视频时长校验**
        *   `if video_length < 0.1:`
        *   `st.info("视频时长不足，请至少输入0.1分钟。")`
        *   `st.stop()`
3.  **加载指示器：**
    *   `with st.spinner("正在生成脚本，请稍候..."):`
    *   用于在长时间运行任务时显示“加载中”动画，提升用户体验。
    *   `generate_script` 函数的调用应放在此 `with` 块的缩进内部。
4.  **调用核心功能：**
    *   `wiki_result, title, script = generate_script(subject, video_length, creativity, openai_api_key)`
    *   传入网页上获取的用户输入值。
5.  **结果展示：**
    *   **成功提示：** `st.success("脚本生成成功！")`
    *   **视频标题：**
        *   `st.subheader("视频标题:")`
        *   `st.write(title)` (显示生成的标题)
    *   **视频脚本：**
        *   `st.subheader("视频脚本:")`
        *   `st.write(script)` (显示生成的脚本)
    *   **维基百科结果 (可折叠)：**
        *   `with st.expander("维基百科搜索结果"):`
        *   `st.write(wiki_result)`
        *   `st.expander()`：用于创建可折叠/展开的内容区域，避免页面过长。
