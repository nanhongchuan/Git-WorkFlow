# 57-AI大模型API用法示例 文本翻译：全语种翻译器

## 1. 主题与目标
- 探索用 AI 大模型 API 实现文本翻译，打造“全语种翻译器”。
- 自动识别源语言，翻译为中文，并尽量保留原文语气，便于业务沟通（如外贸场景）。

## 2. Prompt 设计要点
- 指令内容：
  - 自动判断消息的源语言。
  - 将消息翻译成中文。
  - 尽可能保留原文的语气和情绪。
- 输出格式：
  - 同时给出“原始消息”和“翻译后的消息”。
- 提示工程最佳实践：
  - 用符号将上下文包裹，与指令区分，帮助模型识别“指令 vs 上下文”。
 
## 3. 实现流程（代码思路）
- 使用 `input()` 获取用户输入的原始消息。
- 发送翻译请求：
  - 调用接口函数（文中示例为 get_openai_response 类似的方法）。
  - 传入：模型参数、提前写好的提示词。
  - 用 `format` 将动态的 `message` 变量填入到 `messages` 中。
- 运行并查看结果。

## 4. 示例现象
- 测试葡萄牙语、俄语等消息，能自动识别语言并翻译为中文。
- 翻译能体现原文语气（例如读起来较为强硬的语气也能保留）。

## 5. 能力拓展
- 不只限于自然语言之间的翻译：
  - 也可做编程语言之间的“翻译”（如将一种语言转为另一种），仅需对 Prompt 做相应修改。
- 其他可用场景示例：
  - 文本信息提取：识别地点、人名、公司名等实体。
  - 语气转换：如将口语表述转为书面表达。

## 6. 本章覆盖的 API 基础用法
- 文本总结
- 文本撰写
- 文本分类
- 文本翻译

## 7. 方法论与目标
- 不止做“API + 提示词”的小工具，更要构建完整、实用的 AI 应用。
- 类比：不仅学“垒砖块”，而是要“建房子”。
